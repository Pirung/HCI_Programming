### 픽셀과 해상도
    - 픽셀: 디지털 이미지를 이루는 네모 모양 작은 점
    - 해상도: 정밀도를 나타내는 지표, 픽셀 단위
    - 디스플레이 해상도 PPI, 인쇄물 해상도 DPI
    - 같은 면적이면 픽셀 수가 많으면 해상도 높아 정교
    - 1인치에 표현되는 픽셀수가 많으면 메모리 용량 많이 필요
    - 하나의 픽셀에는 색상 정보 저장
    - 픽셀의 색상 수는 픽셀을 구성하는 비트 수로 결정

### 디지털 이미지의 표준
    - 비트맵 방식
      - 바둑판 모양의 미세한 화소 단위 분해, 각 화소 색상과 위치 저장
      - 일반적 컴퓨터에서 사용되는 디지털 이미지
      - 비트맵 방식의 특징
        - 표현 구조가 단순해 빠르게 화면 띄움
        - 깊이 있는 색조, 부드러운 질감, 상세한 명암 색상
        - 추가 정보를 가지고 있어 활용도가 넓고 호환성 뛰어남
        - 용량이 크고 확대시 화질 저하 단점
      - 엘리어싱 현상 해결 방법
        - 임의의 픽셀과 인접한 4개의 픽셀들에 대해 선형 보간법을 수행
        - 모니터의 해상도 높이기
        - 안티앨리어싱: 경계면의 밝기를 조절해 중간 색상으로 부드럽게 처리
        - 클리어타입 매뉴얼 힌팅: 작업으로 글꼴 크게 확대, 계단현상 왜곡없이 저장
        - 벡터 형식의 특징
          - 수학적인 공식으로 표현해 산술적인 데이터로 저장
          - 벡터 이미지는 평면 또는 공간 상에 특정 좌표값으로 표현
          - 비트맵 이미지는 벡터 이미지로 변환 가능

### 이미지 표현을 위한 컬러 모델
    - 컬러 모델 개요
      - 색은 전기 신호, 구성하는 서로 다른 주파수 가짐
      - RGB 가시 광선을 여러 색깔로 인식
      - 가시광선, 적외선, 자외선으로 구분
      - 가시광선: 사람 눈으로 볼 수 있음
      - 사람 눈의 구성
        - 홍채: 빛 양 조절
        - 수정체 : 초점 맞추기 위해 거리 조절
        - 간상세포: 명암 인식, 어두운 환경 반응
        - 원추세포: 색상 인식, 밝은 환경 반응
      - 망막세포 자극 -> 시각세포에서 전기적 신호를 만들어 신경망 통해 뇌 전달
      - 색 체계: 색의 상대적인 위치와 상호 관계를 2차원 평면과 3차원 공간 표시
      - 컬러모델: 컴퓨터 환경에서 효율적인 커러 영상 처리를 목적 개발

### RGB / CMYK 모델
    - RGB
      - RGV 색 정의, 색 혼합, 보색: 두 가지 색을 겹처 흰색이 나타남
    - CMYK
      - 물체에 반새된 빛의 색상 모델
      - 색의 3원색 빨, 파, 노 로 정의
      - 감산 혼합
    - HSV / HSL 모델
      - 색상
        - 원색, 색상환: 원으로 표현
        - 색상값 H: 가시광선의 스펙트럼을 색상환으로 나타내면 0도 기준으로 상대적인 배치 각도 의미
        - 색상환에서 서로 반대편에 위치한 색은 보색
      - 채도
        - 색의 선명한 정도
        - 색의 맑고 탁함, 강약, 순도
        - 채도값 S: 특정한 색의 가장 진한 상태를 100프로로 하고 흰색이 혼합되는 정도에 따라 감소
        - 채도가 높으면 선명, 반대면 탁함 0%는 무채색
      - 순도 명도
        - 색의 순수한 정도, 밝은 정도
        - 순도값 V 또는 명도값 B는 검은색 0, 가장 밝은 흰색 10
        - 무채색: 명도만 있는 색
      - HSV 컬러 모델
        - 색상, 채도, 순도
        - 0~360도 사이를 일정한 각도로 분할 표현
        - 0 = 빨강, 120 = 초록색 240 = 파랑색
        - 채도는 0~1
      - HSL 컬러 모델
        - 밝은 색, 어두운 색 공간에서 서로 대칭적으로 표현된다는 장점
        - 컴퓨터 그래픽, 포토샵 등 대부분의 이미지 처리 툴에서 많이 사용
      - 컬러볼륨
        - 색 재현율
          - 디스플레이에서 색을 표현할 수 있는 능력을 수치
          - 가능한 많은 색을 표현 가능하다면 실제 환경과 매우 유사하게 표현
        - CIE 1931
          - 1931년에 제정 표준색의 측정 시스템
        - 컬러볼륨
          - UHD HDR 등장하면서 색 재현율의 기준과 목표도 변경
          - 빛에 의해 순간마다 변하는 색 차이를 정교하게 표현하는 정도
          - 빛에 의해 달라지는 색상을 표현하는 능력이 TV 기술에 쟁점
          - 높은 TV는 색 영역이 넓어 원색에서 무채색까지 광범위하게 색상 생성 정확하게 표현

### RAW 이미지 형식
    - RAW 개념
      - 디카, 스캐너 이미지 센서가 촬용한 정보
      - 이미지 센서에 의해 감지된 빛의 정보를 그대로 저장
      - 여러가지 설정 변경 가능
    - 장점
      - 화질 우수, 원본 형태 유지, 사용자 원하는 색 공간 설정 가능
    - 단점
      - JPG 보다 용량 큼
      - 별도의 소프트웨어 필요

### 비트맵 이미지 형식
    - 개념
      - GUI 위해 개발, 윈도우 OS 그래픽, 이미지 저장 기본형식으로 사용
    - 구성
      - 비트맵 파일 헤더
      - 비트맵 정보 헤더
      - 비트맵 팔레트
      - 이미지 픽셀 데이터

### JPG 이미지 형식
    - 개요
      - 웹에서 가장 범용적인 이미지 저장 형식
      - 이미지 호환성은 우수, 압축률에 천차만별
    - 압축 기술
      - 무손실 압축 방식
        - 압축된 데이터를 다시 복원시 이미지 정보 보존
        - 압축률은 손실 압축에 비해 떨어짐, 화질 저하 방지 유용
      - 손실 압축 방식
        - 압축 과정 거치면서 왜곡으로 완전히 원래대로 재생 불가능
        - 높은 압축률, 실용적인 복호화로 화질 획득 가능
      - JPG 압축
        - 이미지 색상 부분을 중점적으로 압축
        - 알고리즘
          - 색상 변환 -> 다운 샘플링 -> DCT 변환 -> 양자화 -> 부호화
          - 아날로그 이미지 상태 RAW 데이터를 압축된 데이터로 변환 저장
          - 원래 이미지 복원시 반대로

### GIF 이미지 형식
    - GIF 이미지 형식 개요
      - JPG, PNG, BMP 같이 사용
      - 이미지로 널리 사용
      - 무손실 압축 방식 LZW 알고리즘 사용
      - 압축률이 떨어지지만 크기가 작고 용량이 적은게 좋음
      - 최대 색상수가 256개
      - 가볍고 속도 측면에서 범용성
      - 움짤 문화

### PNG 이미지 형식
    - 1996년 무손실 압축 형식 등장
    - 오픈소스 탄생
    - 특징
      - GIF와 같은 무손실 압축 이미지 규격
      - JPG와 동일한 트루 컬러 표현
      - GIF 품질이 떨어지는 문제 보완
      - GIF 대비 압축 효율이 10~30 정도 향상
      - 모든 장점 다가지고 있음
      - 파일 용량이 크고 애니메이션 기능 별도 제공
      - PNG 형식은 제작된 이미지에 적합

### 모바일 환경 이미지 형식
    - HEIF 이미지 형식
      - JPG 형식 압축률 보완
      - MPEG 개발한 고호율 동영상 압축 형식인 HEVC 에서 분화한 기술
      - 16비트 지원
      - 연속된 여러 이미지, 뎁스 맵 같은 보조 지원
      - HEVC 형식으로 동영상의 정적인 이미지를 저장하는 방식
      - 현재는 호환성 낮음
      - HEIF는 다양한 데이터 형태를 저장하는 일종의 컨테이너
    - BPG 이미지 형식
      - 품질과 용량 문제를 대체하기 위해 탄생
      - 애니메이션은 MP4 품질이면서 적은 용량으로 GIF 형식 제공, HEVC에서 파생
      - 장점
        - 무손실 압축 방식, 다양한 색공간 지원, 높은 압축률, 애니메이션 지원
        - 동영상 직접 재생 가능, 자비스크립트 디코더로 대부분 브라우저에서 사용 가능
      - 단점
        - 로딩시간, 높은 자원 점유율, 이미지 인코딩 형식으로 사운드 지원 X
    - WebP 이미지 형식
      - 구글이 웹 고속화 위해 만듬
      - 무손실/손실 압축 모두 지원
      - 일반적으로 사진 JPG 로고와 아이콘 등 PNG 애니메이션 GIF 사용
      - JPG, PNG, GIF 모두 대체 가능
      - 사진 압축 효과가 높게 나타남

### 이미지 처리의 개요
    - 이미지 처리 기법의 정의, 특징
      - 스캐너 또는 디카로 입력받아 출력할때까지 파일 조작
      - 입력받은 이미지를 2차원적인 평면 신호로 변환
      - 디지털 이미지 처리는 정확, 빠른 처리를 위해 다양한 컴퓨터 기술들이 사용
    - 대표적인 이미지 처리 기법
      - 이미지의 확대, 축소, 회전 등 기하학적 변환
      - 이미지의 명도, 대비 등과 같으 ㄴ색 보정 및 색 변환
      - 둘 이상의 이미지를 결합
      - 이미지의 변형
      - 패턴인식, 글자 인식, 얼굴 인식등 이미지 인식
      - 특징 추출, 이미지 분할 분류

### 기존의 이미지 처리 기술
    - 히스토그램
      - 도수 분포, 이미지 픽셀 분포, 평균값 편차 등 정보 제공
      - 명암 값 분포 표시 등
    - 히스토그램 평활화
      - 문턱치 값: 이미지와 배경 분리 할 수 있는 가장 적절한 값
      - 이미지 밝기 조절 가능
      - 히스토그램 평활화: 이미지 명암도에 따라 픽셀 개수를 고르게 분포

### 이미지의 콘트라스트와 컨벌루션, 이미지 필터링
    - Contrast
      - 밝음 어두움 차이, 픽셀의 대비도를 높여 뚜렷하게 하거나 윤곽선 검출 가능
    - 컨볼루션
      - 블러링: 이미지를 흐릿하게 표현하는 기법
      - 샤프닝: 이미지 선명하게 하거나 애지를 검출하는데 사용
      - 스무딩: 이미지를 흐리게하거나 노이즈 제거 사용
      - 컨볼루션: 각각의 픽셀에 인접한 주변 픽셀들의 조합으로 대체
    - 이미지 필터링
      - 특정한 마스크 이용 필터링
      - 필터는 사용 목적에 따라 여러가지 형태

### 컴퓨터 비전
    - 인공지능의 한 분야, 인간의 시각적인 인식 능력 컴퓨터에 재현
    - 이미지 또는 동영상에서 특정 객체의 위치 형태 상황 인식
    - 머신 비전, 공장에서 불량 검출이나 제품 검사
    - 카메라가 다양한 스마트 디바이스에 장착 -> 다양한 분야

    - 딥 컴퓨터 비전 기술
      - 인공지능 기술이 비약적으로 발전, 컴퓨터 비전 기술 크게 발전
      - 전통적인 방법
        - 픽셀을 분할해 분석, 평균값을 계산해 이미지 인식
        - 분석 학습을 거친 이미지의 의미에 대해 태그를 부여해 작업을 사람이 별개로 진행
      - 이미지 인식에 딥러닝 적용, 동일하게 평균값 계산
      - 계속되는 이미지 학습을 통해 찾아내거나 추가
      - 태그를 부여하는 작업 절차가 하나의 프로세스로 컴퓨터를 통해 이뤄지므로 작업시간 짧아짐
