### 텍스트의 개념과 특징
    - 텍스트: 여러 문장이 모여 만들어진 문장의 집합, 기억 용량을 적게 차지

### 문서편집기와 워드 프로세서
    - 문서 편집기
      - 문서를 작성할 때 사용, 소스코드, 프로그램 작성
    - 워드프로세서
      - 작성, 편집 저장 및 인쇄 소프트웨어
      - 문서를 손쉽게 작성할 수 있게 다양한 편의 기능 제공
      - 기능 다양

### 코드 시스템
    - 컴퓨터 내부에서 모든 문자는 이진코드로 인코딩
      - 인코딩: 컴퓨터에 저장, 통신에 사용, 문자나 기호를 다른 형식 변환
    - 1963년 아스키 코드
    - 유니코드: 1995년 표준 제정, 각 나라 코드 시스템을 하나로 통합
    - 아스키 코드: 가장 기본적이고 세계적으로 널리 사용, 데이터 처리 및 통신시스템
    - EBCDIC: 
      - IBM에서 사용하기 위해 제작
      - 8비트 사용, 아스키랑 다름
      - 2개의 영역으로 나눠 상위 4비트 하위 4비트 구분
      - 상위 4비트 = 존비트, 하위 4비트 = 디지트 비트 OR 뉴메릭 비트
      - 영역에서 읽은 비트는 코드 테이블 대응 후 문자 해석
      - 150개 정도만 사용
    - ASCII
      - 패리티 비트
        - 8비트로 구성, 공백으로 남는 나머지 1비트는 패리티 비트
        - 오류 검출을 목적으로 사용, 짝수 패리티 비트, 홀수 패리티 비트
          - 한 비트에 에러가 발생하면 오류 검출 가능
          - 패리티 비트를 이용하면 오류 발견 가능, 해결 불가능
          - 한 비트 오류만 검출, 2개 이상은 해결 X
    - 한글 코드
      - 한글의 경우 8비트 코드 체계로 표현 불가
      - 조합형 코드: 초성,중성,종성에 대한 코드 저장 실제 출력때 조합
      - 완성형 코드: 초성, 중성, 종성의 탄글자들을 미리 조합
      - 최상위 비트: 1은 한글 코드 2바이트, 0은 아스키 코드 1바이트
    - 유니코드:
      - 산업체 표준, 소프트웨어의 국제화에 기여, 8비트 단위로 표현
      - 데이터 용량을 많이 차지: 문서 교환, 통신 분야

### 폰트
    - 폰트의 개요
      - 세리프, 산세리프로 분류
      - 문자의 서체, 크기, 위첨자 등 글자 속성 포함
    - 한글 폰트 생성, 진화
      - 굴림체: 1970년대 일본 나루체 모방
      - 맑은 고딕: 한국적인 아름다움과 조형미를 현대적으로 표현, 가독성 극대화
    - 디지털 시대의 텍스트 위기
      - 텍스트에 대한 이해력 저하
      - 위기의 한글 사용
      - 디지털 시대와 손글씨의 차이

### 문서 시장의 변화
    - 문서 표준: 지속적인 새로운 버전의 소프트웨어 제공, HWP,DOC 등 특정 기업에 종속된 폐쇄형 전자문서
    - ODF 
      - 개방형 문서 표준
      - 종이 문서 사용량 감소
      - 폐쇄형 전자문서보다 품질이 떨어짐
    - 클라우드 환경 오피스 서비스
      - 최근 오피스 소프트웨어는 PC 중심 -> 모바일과 클라우드 환경
      - 별도의 프로그램 설치 없이 모든 환경 연동
      - 문서를 클라우드에 저장 후 모든 플랫폼에 공유 편집 가능
    - PDF
      - 개방형 문서 표준 ODF 기반 전자문서
      - 문서 위변저 방지 기능-> 열람, 보관용 가장 적합
      - 작성 프로그램인 아크로뱃 유로
      - 뷰어 아크로뱃 리더는 무료
      - 주요 전자책 포맷 각광
      - 장점:
        - 문서 호환성이 높고 파일의 무결성을 가짐
          - 파일의 무결성, 정보를 공유할 때 적합한 형식, 컴퓨터에서 읽기, 인쇄 소스파일 그대로 유지
        - 문서 사용과 관리가 편리
        - 문서 보안 뛰어남
        - 쌍방향 인터페이스 삽입 가능
      - 단점
        - 편집 불편
        - 문서의 제작 방식이 동일하지 않고 다양한 파일 형식이 존재
        - 모니터 가독성 낮음
      - DRM 기술 도입
        - 무단사용 방지 기술, 특정 인물만 접근, 정해진 시간 동안만 접근
        - 캡쳐 불가능
        - 지정된 PC와 스마트폰 등에서 제한적으로 사용 가능
        - 기밀 서류 보관, 데이터페이스를 구축 사용

### 글자 입력 기술
    - 동작 인식, 음성 인식 방식
    - 다양한 터치스크린 방식
      - 시크릿 팔찌: 피코프레젝터를 통해 손가락 움직임 감지
      - 버드: 손가락에 끼우면 어떤 사물이든 터치 동작 인식, 음성 인식 가능

### 문자 인식 기술
    - 기존의 인쇄 자료, 손으로 기록한것을 컴퓨터 자동 인식
    - OCR 이라 부름
    - 패턴 정합: 입력된 문자와 컴퓨터에 기억된 문자 유사성 정확도 이용, 인쇄 문자 인식, 난이도는 숫자, 영문자, 기호, 한글, 한자 순
    - 구조 분석: 고유 특정적인 선의 형태와 특성에 의해 식별
    - 문자 인식 순서: 기존 인쇄 자료를 스캐너,카메라 통해 이미지 형태로 읽음
    - 데이터 내용을 분석, 그림, 글자 영역 구분, 텍스트로 변환
    - OCR + 인공지능
      - 반복되는 검증 업무 감소
      - 번역 이미지 검색, 텍스트 분석, 챗봇, 

### 전자책
    - 디지털 콘텐츠로 제작된 파일 형식 책
    - 스마트폰, 태블릿
    - 아직도 빠르게 성장 불가능
